# Короткий план защиты

## Задача

Компания получает много писем в IT-поддержку. Нужно автоматически разложить их по категориям, чтобы важные инциденты быстрее попадали к ответственным людям, а спам и нестандартные файлы не мешали обработке.

## Архитектура

Пайплайн:

```text
inbox/ -> MailReader -> RuleBasedClassifier -> MailProcessor -> sorted folders + logs
```

`MailReader` отвечает только за чтение и обработку форматов.  
`RuleBasedClassifier` отвечает только за выбор категории.  
`MailProcessor` связывает всё вместе, работает с папками и пишет отчёты.

## Устойчивость

Система не падает на неожиданных данных:

- пустые письма идут в `empty_mail`;
- бинарные файлы и изображения идут в `attachments_or_binary`;
- повреждённый JSON получает статус `malformed_json`, но всё равно анализируется как текст;
- неизвестные письма идут в `needs_review`.

## Почему не нейросеть

Для учебного проекта и защиты важна объяснимость. Правила с весами легко показать комиссии: видно, почему письмо попало в категорию, какие слова сработали и какой score получился.

## Что можно улучшить

- добавить больше правил после анализа новых писем;
- хранить правила в отдельном YAML/JSON-файле;
- добавить веб-интерфейс для ручной проверки `needs_review`;
- заменить или дополнить правила ML-моделью после накопления размеченных данных.
